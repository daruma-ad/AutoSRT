# AutoSRT ワンショット開発・完全再構築プロンプト

以下は、この「AutoSRT」という複雑なアプリケーション全体（UI、音声解析、XML処理、LLM連携、バグ回避のガードレールすべて）を、新しいAIや別プロジェクトの環境において**「一気通貫（ワンショット）」で全く同じレベルのものを出力させるためのマスタープロンプト**です。

この内容をコピーして強力なLLM（Gemini 2.5 ProやClaude 3.5 Sonnetなど）に投げるだけで、同等品質のコードベース全体を再構築させることができます。

# 1. プロジェクト概要と技術スタック

- **UIフレームワーク**: Python + Streamlit
- **音声解析**: ffmpeg-python, numpy, scipy

# 2. ファイルごとの要件定義

## ① `app.py` (メインUIとルーティング)
- **レイアウト**: Streamlitを用い、左サイドバーにGemini APIキー設定と、カット処理のパラメータ（閾値、最小無音時間、パディング）スライダーを配置。
- **メイン画面**: 「FCP XML 無音カット」と「SRT 字幕校正」の2つのタブ構成。
- **UI UX**:
  - カット設定において「⚡ジェットカット」「🎙️スタンダード」「🎬シネマティック」の3段階のプリセットボタンを実装し、押すと連動してスライダー値が変わること。
  - スライダーの下に「29.97fpsなら何フレームか」をリアルタイムでテキスト換算表示すること。
- **ファイル保存機能**: Streamlitの標準ダウンロードボタンに加え、処理済みファイルをMac等のPCの「デスクトップ」に直接書き出す（`pathlib.Path.home() / "Desktop"`）独自処理である `_save_to_desktop()` ボタンを実装すること。

## ② `audio_processor.py` (音声解析とXML再構築)
- **無音検出アルゴリズム**: 音声をWAVテンポラリに変換後、50msウィンドウで分割してRMS音量を算出。最大音量に対するパーセンテージで無音区間を判定し、パディング（余白）を適用して有音区間リストを作成すること。
- **FCP XMLパースと再構築**: 読み込んだXMLの `<sequence>/<media>/<video>/<track>` 内の `<clipitem>` をコピーし、先ほど算出した有音区間のリストに合わせて分割・再構築すること。
- **【絶対厳守課題：NTSCドリフト防止】**: 
  - NTSC（29.97fps等）の場合、XMLの `timebase`（30）をそのまま使わず、必ず `timebase * 1000 / 1001` で真のFPSを算出すること。
  - Premiere固有の `pproTicks` （1秒＝254,016,000,000）を計算する際は、長尺動画での誤差蓄積を防ぐため、一度「フレーム境界の秒数」を正確に算出し、それに 254016000000 を掛けて四捨五入するアプローチを徹底すること。

## ③ `srt_processor.py` (AI字幕校正と究極のガードレール)
- **LLM連携**: `pysrt` でパースした字幕エントリを、チャンク（基本30単位）に分割し、システムプロンプト（ユーザーの文脈情報、専門用語リストを含む）と共に Gemini API に送信し、元のフォーマット（番号、タイムスタンプ）を維持したままテキスト校正を行わせること。
- **スマートフォーマッティング機能**: AI送信前の前処理として、極端に短い字幕（数文字程度）は次の字幕へ結合し、1行あたりの推奨文字数に達したら改行を入れ、最大2行に制限するアルゴリズムを実装すること。
- **【絶対厳守課題：タイムコード崩壊を防ぐ多重ガードレール】**:
  LLMはプロンプトの指示を破る前提で、以下の2つの防壁を必ず実装すること。
  1. **正規表現オートリペア**: LLMが改行を忘れ、テキストの末尾に次のエントリ番号を繋げる（例: `文章281\n00:09:`）バグを防ぐため、パース直前に `re.sub(r'([^\n\d])(\d+\s*\n\d{2}:\d{2}:\d{2}[,\.]\d{3}\s*-->)', r'\1\n\n\2', content)` を適用し、強制的に改行を補入すること。
  2. **オフセット逆算マッピング**: LLMが字幕をいくつか削除し、なおかつ番号を「1」から振り直した場合、位置情報で同期させると甚大な数秒単位の音声ズレが起きる。AIが返した `start` レコードの先頭8文字（`HH:MM:SS`）を使って原文とのズレ幅（オフセット）を特定し、AIが付与したインデックス番号を強制的に「正しい連番」へ上書きすること。照合不能時は処理を破棄し、安全な原文を採用するフォールバックを入れること（絶対にタイムコードと音声の同期を優先）。
