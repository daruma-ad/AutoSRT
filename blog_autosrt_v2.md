# Antigravity×Premiere Pro：動画編集の「最もツラい作業」を消し去るAI自動カット＆テロップ校正ツール

動画編集の案件をこなす中で、**一番時間がかかって、一番心が折れる作業**は何でしょうか？

エフェクトの作り込み？ 色調補正？ 音の調整？
いいえ、違いますよね。

**「ひたすら無音を削るジェットカット」と「誤字脱字だらけのテロップ修正」です。**

もちろん、今のAdobe Premiere Proには優秀な自動文字起こし機能や、無音を自動検出する機能も備わっています。「じゃあそれで十分じゃないか」と思うかもしれません。
しかし、現場でゴリゴリ編集している人なら分かるはずです。

*   **「Premiereの自動カットは、微妙な『間』の調整が直感的にやりづらい…」**
*   **「文字起こしは便利だけど、同音異義語の誤変換が多すぎる。結局1文字ずつ目視で直して、文節の改行位置を整えるのに膨大な時間が溶ける…」**

「それならいっそ、最先端のAIエージェントとコードベースの動画生成（Remotion等）を組み合わせて、全て全自動で作ってしまえばいいのでは？」
実際、私もそれを試しました。しかし結論から言うと、**現段階での完全自動化は実用的ではありませんでした。** クライアントの案件ごとに求められるテロップの細かい表現、演出方法、絶妙なタイミングの微調整において、やはりPremiere Proというソフトは**神がかったUIを持つ超優秀なツール**なのです。ゼロからコードで表現をチューニングしていくよりも、Premiereのタイムラインでマウスを握る方が圧倒的に効率的でクリエイティブです。

つまり、私たちが本当に求めているのは「編集ソフトを乗り換えること」ではなく、**「Premiere Proの強みを活かしつつ、一番面倒な『カット』と『テロップ修正』という泥臭い作業だけを、最新のAIに完璧に丸投げすること」**でした。

そして今、最新のAIモデル（Gemini 2.0 Flash等）の精度向上により、それがついに可能になりました。

今回は、強力なAIアシスタント**「Antigravity」**の力を借りて、この理想を叶えるWebアプリケーション**「AutoSRT」**を開発しました。

Premiere Proを使っていて、AIの力で己の編集限界スピードを突破し、より多くの案件をこなしていきたいクリエイターにとって、**間違いなく導入すべきシステムアプリ**です。本記事では、このツールの圧倒的な威力と、その開発の裏側（完全なソースコード）を大公開します！

## 【無料公開部分】AutoSRTでできること

このアプリは、あなたのPremiere Proでの編集前処理を「数クリック」で終わらせます。

### 1. Premiere Proに直接読める「AI自動カット（FCP XML生成）」

音声ファイル（MP3/WAV）と、書き出したFCP XMLをツールに読み込ませるだけで、AIが波形を解析して不要な無音部分を自動でカット。Premiere Proでそのまま読み込める新しいXMLを吐き出してくれます。

さらに、動画のテイストに合わせて**3つのカットスタイル**をワンクリックで選べるようにしました。

*   **⚡ ジェットカット**: テンポ重視。無音をほぼ全て除去し、テキパキした印象に（YouTube・解説動画向け）。
*   **🎙️ スタンダード**: 自然な「間」を少し残すバランス型（インタビュー・1人語り向け）。
*   **🎬 シネマティック**: 余韻や沈黙を大切にする控えめなカット（ドキュメンタリー・感動系向け）。

「最小無音時間」や「パディング（余白）」といった細かなパラメータもスライダーで直感的に微調整可能。しかも「29.97fpsなら何フレームか」がリアルタイム表示されるため、Premiereのタイムライン感覚のまま数値を設定できます。

### 2. Gemini 2.0 Flashによる「超・文脈考慮型」テロップ校正

ここが既存ツールとの決定的な違いです。
元のSRTファイルには、「同音異義語のミス」や「専門用語の誤変換」がつきものです。

AutoSRTでは、Googleの最新・超高速AIモデル「Gemini 2.0 Flash」を活用。「動画のジャンル」や「専門用語リスト」を入力しておくことで、AIが動画の**文脈を理解**し、**タイムスタンプ（字幕の表示タイミング）を一切壊さずにテキストのみを完璧に校正**してくれます。

例えば「マイナス金利」や「プライマリーバランス」といった専門用語も、事前にリスト化しておけばAIが文脈から判断して一発で正確な表記に書き換えます。もう、タイムライン上で文字を1つ1つ打ち直す必要はありません。

### 3. 直感的なUIと「デスクトップ保存」機能

PythonのWebフレームワーク「Streamlit」を採用し、ブラウザ上で直感的に操作できる美しいUIを構築しました。
処理が終わったら、「📥 ダウンロード」ボタンだけでなく**「🖥️ デスクトップに保存」**ボタンも用意。Macのデスクトップに直接ファイルが保存されるので、ダウンロードフォルダをごちゃごちゃ探すストレスもありません。そのままPremiereにドラッグ＆ドロップするだけです。

### 1. FCP XML再構築アルゴリズムのキモ

Premiere ProのFCP XMLには動画のクリップ情報（開始時間、終了時間、イン点、アウト点）が記述されています。
無音カットを行うには、音声波形から「有音セグメント（保持したい部分）」のリストを作成し、それに合わせてXML内の`<clipitem>`を分割・再計算する必要があります。

特に重要なのは以下のパラメータでした：
1.  **フレームレートのマッチング**: 29.97fps（NTSC）等のタイムベースを正確に計算すること。
2.  **パディング（余白）**: 発話の頭切れ・尻切れを防ぐための「遊び」です。カットポイントの前後に`00;00;00;02`（2フレーム）程度の余白を持たせます。

```python
# 音声解析のコア部分（イメージ）
def process_silence_cut(audio_bytes, xml_bytes, threshold_pct, min_silence_ms, padding_ms):
    # 1. 音声をWAVに変換しRMS（音圧）を計算
    # 2. 閾値以下の区間を「無音」としてリスト化
    # 3. 指定したミリ秒（ms）に基づいてパディングを追加
    # 4. XMLをパースし、有音区間に合わせてclipitemを再構築
    ...
```

本アプリでは、このミリ秒の指定をスライダーで細かく（1ms単位）設定できるようにし、さらに`67ms -> 2フレーム(29.97fps)`といったリアルタイム換算表示をStreamlitで実装し、エディターが直感的に使えるようにしています。

### 2. Gemini APIによる「壊れない」テロップ校正

SRTファイルをLLMに食わせる際、最大の難関は**「AIがタイムスタンプを勝手に変えたり、独自フォーマットで返してきてファイルが致命的に壊れる」**ことです。

これを防ぐため、以下のプロンプトエンジニアリング（AIへの指示の工夫）を行っています。

*   一度に全て送らず、**チャンク（例：30行ごと）に分割**して送信する。
*   プロンプトで**「フォーマットの厳密な死守」**を絶対命令とする。

```python
system_prompt = f"""
あなたは優秀なプロの映像翻訳・字幕校正者です。
入力されたSRT字幕データのテキスト部分のみを、文脈に合わせて自然な日本語に校正してください。

【厳守事項】
1. タイムスタンプや連番は絶対に元のまま維持すること。
2. SRTフォーマットを崩さないこと。
3. テキスト以外の余計な解説は一切出力しないこと。

【文脈情報】
{context}

【専門用語リスト】
{terminology}
"""
```
Gemini 2.0 Flashモデルは非常に安価で超高速です。動画1本分（数百チャンク）のテキストを往復させても数円〜十数円程度しかかからず、実用コストとして全く問題になりません。

### 3. デスクトップ直接保存の実装

ブラウザ利用時の「ダウンロードフォルダに保存されたファイルを探すのが面倒」という地味なストレスは、Pythonの`pathlib`を使って解決しています。Webアプリとしてではなく、ローカル上で動かす社内ツールアプローチだからこそできる力技です。

```python
import pathlib
import streamlit as st

def _save_to_desktop(content: bytes, filename: str):
    desktop = pathlib.Path.home() / "Desktop"
    save_path = desktop / filename
    with open(save_path, "wb") as f:
        f.write(content)
    st.success(f"✅ デスクトップに保存しました: {filename}")
```
