# Antigravity×Premiere Pro：動画編集を劇的に時短する「AI自動カット＆テロップ校正ツール」の作り方

動画編集において、地味で最も時間がかかる作業といえば**「無音部分のカット（俗に言うジェットカット）」**と**「テロップの誤字脱字修正」**ではないでしょうか？

特に1人語りやインタビュー動画では、この「前処理」だけで数時間溶けてしまうことも珍しくありません。

今回は、強力なAIアシスタント**「Antigravity」**の力を借りて、この面倒な前処理を全自動化するWebアプリケーション**「AutoSRT」**を開発しました。
動画の音声を解析してPremiere Pro用の「カット済みXML」を生成し、さらに最新の「Gemini 2.0 Flash」APIを使ってSRT字幕を文脈に合わせて自動校正してくれる画期的なツールです。

本記事では、このツールの魅力と、実際のコードを含めた開発の舞台裏を大公開します！

## 【無料公開部分】AutoSRTでできること

まずは、今回開発したツールでどのようなことができるのか、その主要な機能をご紹介します。

### 1. Premiere Proに直接読める「AI自動カット（FCP XML生成）」

音声ファイル（MP3/WAV）と、書き出したFCP XMLをツールに読み込ませるだけで、AIが波形を解析して不要な無音部分を自動でカット。Premiere Proでそのまま読み込める新しいXMLを吐き出してくれます。

さらに、ユーザーの用途に合わせて**3つのカットスタイル**をワンクリックで選べるようにしました。

*   **⚡ ジェットカット**: テンポ重視。無音をほぼ全て除去し、テキパキした印象に（YouTube・解説動画向け）。
*   **🎙️ スタンダード**: 自然な「間」を少し残すバランス型（インタビュー・1人語り向け）。
*   **🎬 シネマティック**: 余韻や沈黙を大切にする控えめなカット（ドキュメンタリー・感動系向け）。

「最小無音時間」や「パディング（余白）」といったパラメータも、スライダーで直感的に微調整可能。しかも「29.97fpsなら何フレームか」がリアルタイム表示されるため、プロの映像クリエイターにも納得の仕様です。

### 2. Gemini 2.0 Flashによる「超・文脈考慮型」テロップ校正

既存の文字起こしツール（Whisperなど）で出力したSRTファイルには、「同音異義語のミス」や「専門用語の誤変換」がつきものです。

AutoSRTでは、Googleの最新超高速AIモデル「Gemini 2.0 Flash」を活用。「動画のジャンル」や「専門用語リスト」を入力しておくことで、AIが動画の**文脈を理解**し、タイムスタンプを一切壊さずにテキストのみを完璧に校正してくれます。

「マイナス金利」や「プライマリーバランス」といった専門用語も、事前にリスト化しておけばAIが文脈から判断して一発で正確な表記に修正します。

### 3. 直感的なUIと「デスクトップ保存」機能

PythonのWebフレームワーク「Streamlit」を採用し、ブラウザ上で直感的に操作できる美しいUIを構築しました。
処理が終わったら、「📥 ダウンロード」ボタンだけでなく**「🖥️ デスクトップに保存」**ボタンも用意。Macのデスクトップに直接ファイルが保存されるので、ダウンロードフォルダを探し回るストレスもありません。

### 1. FCP XML再構築アルゴリズムのキモ

Premiere ProのFCP XMLには動画のクリップ情報（開始時間、終了時間、イン点、アウト点）が記述されています。
無音カットを行うには、音声波形から「有音セグメント（保持したい部分）」のリストを作成し、それに合わせてXML内の`<clipitem>`を分割・再計算する必要があります。

特に重要なのは以下のパラメータでした：
1.  **フレームレートのマッチング**: 29.97fps（NTSC）等のタイムベースを正確に計算すること。
2.  **パディング（余白）**: 発話の頭切れ・尻切れを防ぐため、カットポイントの前後に`00;00;00;02`（2フレーム）程度の余白を持たせること。


本アプリでは、このミリ秒の指定をスライダーで細かく（1ms単位）設定できるようにし、さらに`67ms -> 2フレーム(29.97fps)`といったリアルタイム換算表示をStreamlitで実装しています。

### 2. Gemini APIによる「壊れない」テロップ校正

SRTファイルをLLMに食わせる際、最大の難関は**「AIがタイムスタンプを勝手に変えたり、独自フォーマットで返してきてファイルが壊れる」**ことです。

これを防ぐため、以下のプロンプトエンジニアリングを行っています。

*   一度に全て送らず、**チャンク（例：30行ごと）に分割**して送信する。
*   プロンプトで**「出力フォーマットの厳密な死守」**を命令する。

```python
system_prompt = f"""
あなたは優秀なプロの映像翻訳・字幕校正者です。
入力されたSRT字幕データのテキスト部分のみを、文脈に合わせて自然な日本語に校正してください。

【厳守事項】
1. タイムスタンプや連番は絶対に元のまま維持すること。
2. SRTフォーマットを崩さないこと。
3. テキスト以外の余計な解説は一切出力しないこと。

【文脈情報】
{context}

【専門用語リスト】
{terminology}
"""
```
Gemini 2.0 Flashは非常に安価で高速なため、動画1本分（数百チャンク）を処理しても数円〜十数円程度しかかからず、実用性が極めて高いです。

### 3. デスクトップ直接保存の実装

ブラウザ利用時の「ダウンロードフォルダに埋もれる」問題は、Pythonの`pathlib`を使って解決しています。Streamlitはデフォルトではブラウザの機能を使いますが、ローカルで動かすツールであれば直接ファイルシステムに書き込めます。

```python
import pathlib
import streamlit as st

def _save_to_desktop(content: bytes, filename: str):
    desktop = pathlib.Path.home() / "Desktop"
    save_path = desktop / filename
    with open(save_path, "wb") as f:
        f.write(content)
    st.success(f"✅ デスクトップに保存しました: {filename}")
```
