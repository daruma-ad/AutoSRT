# AutoSRT 開発・運用記録（パーソナル・ログ）

このドキュメントは、動画編集前処理ツール「AutoSRT」の開発プロセス、直面した「絶望的な課題」、そしてそれをどのように「技術の力」でねじ伏せたのかを記録した個人用ログです。

---

## 📅 2026年4月7日：プロジェクト開始とコア機能構築

### 1. 初期コンセプトと技術選定
- **目的**: 編集作業で最も辛い「無音カット」と「字幕校正」を自動化し、Premiere Proの作業時間を劇的に短縮する。
- **技術スタック**: Python + Streamlit + FFmpeg + Gemini API。
- **背景**: 「完全自動化（Remotion等）」の夢を追うのではなく、微調整のしやすさにおいて「神ツール」であるPremiere Proに渡すための**「80%を最速で終わらせる前処理」**に特化する現実的な戦略。

### 2. インテリジェント無音カットの実装
- **課題**: 音量閾値だけでは、発話の頭切れが発生する。
- **解決策**: 
  - 50ms単位のRMS解析と、ユーザーがミリ秒単位で調整できるUI（パディング機能）を構築。
  - Premiere独自の絶対時間 `pproTicks` （1秒＝254,016,000,000）を正確に計算し、FCP XMLを再構成するロジックを確立。

## 🛠️ 機能拡張とUI/UXのブラッシュアップ

### 3. 編集者目線の操作向上
- **改善**: 
  - 3段階のカットプリセット（ジェットカット / スタンダード / シネマティック）を導入。
  - スライダーの下に「29.97fpsなら何フレームか」をリアルタイム換算表示。

### 4. デスクトップ直接保存機能
- **改善**: ダウンロードフォルダに埋もれるストレスを解消するため、Macのデスクトップに直接 `_save_to_desktop` するネイティブ連携を実装。

### 5. スマートフォーマッティング（テレビ品質の前処理）
- **課題**: LLMで文字を修正するだけでは、「短すぎる字幕」や「画面を覆い尽くす長文」が残り、結局手作業が発生する。
- **解決策**: 字幕文字数を解析し、「1文字〜数文字の細切れ字幕（あのー 等）」を文脈を壊さずに次エントリへ結合。さらに「1行あたりの推奨文字数」を超えた場合は自動改行し、強制的に最大2行に抑えるスマートアルゴリズムを追加。

## 🔧 2026年4月8日：タイムコードの雪崩を防ぐ「究極のガードレール」実装

Premiere での実地テストにより、AIの気まぐれによる「恐怖のタイムスタンプ崩壊」が次々と発見された。動画編集において「音がズレる」ことは死を意味する。妥協なきガードレールを三段構えで実装。

### 6. 【致命的】NTSC フレーム計算の累積ドリフト（動画後半のズレ）
- **症状**: 長尺動画になるほど、 XML タイムラインと音声がズレていく（10分で0.6秒）。
- **原因**: `sec_to_timebase_frames()` が NTSC（29.97fps）環境であっても公称値の「30」で計算していたため、誤差が蓄積。
- **解決策**: NTSCの場合は `timebase × 1000 / 1001` という厳密なFPS計算を導入し、`pproTicks` もフレーム境界に同期。1時間の動画でも1フレームのズレも許さない精度を実現。

### 7. ガードレール①：LLMの「改行忘れ」への正規表現オートリペア
- **症状**: 画面のテロップに `...給付、281\n00:09:53,526 -->` のように次の字幕のタイムコード自体が表示されてしまう。
- **原因**: LLMが改行を忘れ、前のエントリのテキスト末尾に次のインデックスブロックをそのまま連結させてきたため、パーサーが「文字」として認識した。
- **解決策**: `[^\n\d]`（数字や改行以外の文字）の直後にSRTブロックが来た場合、間に強制的に `\n\n` を挿入する強力なオートリペア（正規表現 `re.sub`）をパース前に挟み込む。

### 8. ガードレール②：AIの「勝手なインデックスリセット」へのオフセット逆算
- **症状**: 一部の字幕と音声が数秒〜十数秒がっつりズレる。
- **原因**: LLMが短すぎるエントリを勝手に削除し、**さらに番号を1から振り直した**場合、位置による照合が破綻。誤ったタイムスタンプが全ての字幕にずれて割り当てられた。
- **解決策**: 位置情報に頼るのをやめ、**LLMが返してきたタイムコードの `HH:MM:SS` 部分** を頼りに「本当の原文位置」を逆算してオフセット（ズレ幅）を特定。インデックスを強制的に正しい番号へ上書きする堅牢なロジックを実装。

### 9. ガードレール③：照合不能時のフォールバック
- **解決策**: 上記手段でも修復できないほどAIがテキストを破壊した場合、「無理にズレた時間を当てるより、その数百文字だけAIの校正を破棄して原文を採用する」という安全装置（フォールバック）を実装。**「同期の完璧さ」を絶対優先**するプロフェッショナル仕様に仕上がった。
