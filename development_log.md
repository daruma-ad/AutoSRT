# AutoSRT 開発・運用記録（パーソナル・ログ）

このドキュメントは、動画編集前処理ツール「AutoSRT」の開発プロセス、直面した課題、そして解決策を記録した個人用ログです。

---

## 📅 2026年4月7日：プロジェクト開始とコア機能構築

### 1. 初期コンセプトと技術選定
- **目的**: 動画編集の「無音カット」と「字幕校正」を自動化し、Premiere Proでの作業時間を劇的に短縮する。
- **技術スタック**: Python + Streamlit + FFmpeg + Gemini API。
- **背景**: 既存の完全自動化ツール（Remotion等）は微調整が難しいため、あえて「最強の編集・プレビュー機」であるPremiere Proに渡すための「前処理」に特化する戦略。

### 2. インテリジェント無音カットの実装
- **課題**: 単なる音量閾値だけでは、ノイズによる誤判定や、発話の頭切れが発生する。
- **解決策**: 
  - 50ms単位のRMS解析をベースに、ユーザーが「最小無音時間」と「パディング（余白）」をミリ秒（ms）単位で調整できるUIを構築。
  - Premiere Pro 独自の `pproTicks`（1秒 = 254,016,000,000）を正確に計算し、FCP XML を再構成するロジックを確立。

### 3. Gemini API への統合
- **変更**: OpenAI / Anthropic を廃止し、最新の `google-generativeai` (Gemini 2.0 Flash) に一本化。
- **利点**: 
  - 字幕校正における圧倒的な文脈理解力。
  - トークン単価が非常に安く、長時間動画でもコストを気にせず利用可能。
  - タイムスタンプを維持するための厳格なプロンプトエンジニアリングを適用。

## 🛠️ 機能拡張とUI/UXのブラッシュアップ

### 4. 編集者目線の操作向上
- **課題**: ミリ秒指定が、動画編集者の「フレーム感覚」と乖離（かいり）している。
- **改善**: 
  - スライダーの下に「29.97fps / 60fps / 24fps」でのフレーム換算値をリアルタイム表示。
  - **3段階のカットプリセット**（ジェットカット / スタンダード / シネマティック）を導入。一発で推奨値をセット可能に。
  - 音量閾値に対して「控えめ / 標準 / 積極的」という動的な補足説明を追加。

### 5. デスクトップ直接保存機能
- **改善**: ブラウザのダウンロードフォルダに埋もれる不便さを解消するため、Macのデスクトップに直接 `_save_to_desktop` する補助機能を実装。
